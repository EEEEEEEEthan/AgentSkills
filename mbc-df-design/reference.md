# 矮人要塞式涌现设计 · 参考

仅在需要展开系统推导时阅读。

## 核心思路

| 原则 | 含义 |
|------|------|
| 规则简单，交互复杂 | 每条规则可读可测，组合后行为不可穷举 |
| 分层模拟 | 物理 / 经济 / 社会 / 叙事各层有独立状态，层间通过事件耦合 |
| 失败即内容 | 挫折产生故事，而非仅惩罚 |
| 信息不完全 | 玩家推理世界，而非读取 UI 全部真相 |
| 历史可追溯 | 世界状态有因果链，最美角落可被事后复盘 |

## 系统推导检查单

推导每个系统时自问：

1. 它直接支撑最美角落的哪条必要条件？
2. 去掉它，最美角落还能涌现吗？
3. 它与现有框架是扩展、替换还是并行？
4. 最小可玩切片（MVP）需要哪几条规则？

## 涌现 vs 脚本化

| 涌现（优先） | 脚本化（慎用） |
|-------------|---------------|
| 规则组合自然产生该时刻 | 唯一触发器 + 过场动画 |
| 玩家事后能解释因果 | 玩家感到「系统替我演了」 |
| 可复现但不固定 | 每次完全相同 |

最美角落应主要来自涌现；若必须用脚本保底，标为冲突交由用户拍板。

## 与最美角落的对齐

```
最美角落必要条件
    ↓ 每条至少一个系统承接
系统核心规则
    ↓ 规则间接口明确
涌现预期
    ↓ 可观测、可测试
验收：玩家能复述那一刻
```
