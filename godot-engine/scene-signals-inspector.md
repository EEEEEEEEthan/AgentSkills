# 场景内静态信号优先在检视器连接

## 约定

同一 `.tscn` 内，发射端和接收端都已存在的静态信号，优先在检视器的信号面板里连接，并写入场景文件的 `[connection ...]`。

## 不推荐

不要默认在 `_ready()` 里补：

```gdscript
timer.timeout.connect(_on_timer_timeout)
```

如果这个连接关系本来就是场景结构的一部分，放在代码里会更隐蔽。

## 这样做的好处

- 连接关系在场景里一眼可见
- 合并冲突时更容易 diff
- 改方法名时编辑器更容易发现断连

## 例外

下面这些情况仍然适合代码连接：

- 运行时动态创建节点
- 连接目标依赖分支逻辑
- 同一节点需要按条件切换连接关系
