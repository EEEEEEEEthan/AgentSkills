# C# API 成本

## 核心原则

Godot C# 调用引擎 API 有跨托管/原生边界成本。

所以默认原则是：

- 少读
- 少写
- 少重复调用

## 高风险点

### `Engine.IsEditorHint()`

这个调用不是“普通静态判断”，它有边界开销。

如果一段逻辑里会多次用到它，不要每次都直接调，应该封装并缓存第一次结果。

## 默认做法

### 1. 缓存查询结果

适合：

- `Engine.IsEditorHint()`
- 一次流程内不会变化的引擎状态
- 同一帧内反复访问的属性

### 2. 只在值变化时写回

适合：

- `Position`
- `Rotation`
- `Scale`
- 相机或节点的其他频繁更新属性

## 推荐模式

用本地缓存字段保存上次值。

setter 或更新逻辑里先判断：

```csharp
if (cachedValue == nextValue)
{
    return;
}
```

只有变化时才：

1. 更新缓存
2. 写回引擎对象

## 适用场景

- 每帧更新角色、相机、UI 节点变换
- 编辑器与运行时分支判断
- 高频轮询引擎对象状态

## 判断标准

如果某段 C# 代码在循环、`_Process()`、`_PhysicsProcess()` 或高频 UI 刷新里频繁访问引擎属性，就要检查：

- 能不能先缓存
- 能不能批量算完再写回
- 能不能避免重复调用同一个引擎查询
