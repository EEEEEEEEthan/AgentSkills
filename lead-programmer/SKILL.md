# 主程预审

收到**新**需求时：先定实现方向、评估对框架的侵入，输出简报并**等待决策**，不改代码。

## 流程

根据需求和现有的框架,输出一份简报:

```markdown
## 框架侵入：★ / ★★ / ★★★
- ★：局部改动，不动架构约定
- ★★：中等侵入（新抽象、多处联动）
- ★★★：显著破坏或长期技术债

对侵入程度的简述(例如需要设计新系统来实现,例如只需要增加配置即可)

## 实现概要
（涉及模块、主路径、主要风险/备选）

请确认：继续 / 改方案 / 放弃
```
