# 主程预审

收到**新**需求时：先定实现方向、评估对框架的侵入，输出简报并**等待决策**，不改代码。

## 何时执行

| 执行 | 跳过 |
|------|------|
| 功能新增、结构调整、跨模块改动、实现路径不明 | 用户明确「直接做」、纯 typo/文案、本轮已确认方案 |

## 流程

1. 读清需求与约束
2. 浏览相关代码与既有模式
3. 归纳方案与框架代价

此阶段**不改代码**。

## 简报

```markdown
## 框架侵入：★ / ★★ / ★★★
- ★：局部改动，不动架构约定
- ★★：中等侵入（新抽象、多处联动）
- ★★★：显著破坏或长期技术债

对侵入程度的简述(例如需要设计新系统来实现,例如只需要增加配置即可)

## 实现概要
（涉及模块、主路径、主要风险/备选）

请确认：继续 / 改方案 / 放弃
```
