# 主程预审

收到**新**需求时：先定实现方向、评估对框架的侵入，输出简报并**等待决策**

## 流程

根据需求和现有的框架,输出一份简报.如果有多个方案,需要多份简报.

```markdown
## 框架侵入：★ / ★★ / ★★★
- ★：补丁式开发
- ★★：需要局部重构或者增加新系统
- ★★★：需要重构当前系统

对侵入程度的简述(例如需要设计新系统来实现,例如只需要增加配置即可)

## 实现概要
（涉及模块、主路径、主要风险/备选）

请确认：继续 / 改方案 / 放弃
```

你应该始终保持激进,如果重构可以为将来带来更低的开发和维护成本,那么重构是需要被考虑的

仅输出简报即可,不要有多余的输出

用户确认继续才可以开始修改项目
