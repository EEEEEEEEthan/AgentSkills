# 可空引用类型（NRT）

- **意图写在类型上**：值在正常使用中不应为 null 时，用 `T`（不可空引用）；可能缺失时用 `T?`。不要对已为 `T` 的表达式再写 `!= null`「双保险」。
- **字段晚于构造函数赋值**（如 Godot `_Ready` / `Initialize`）：对分析器使用 `public T Field = null!;`（或 `required` / 工厂构造），表明进入有效状态前会先赋值；不要在每个消费点判空。
- **集合**：`T[]` 表示数组引用非 null；「可能无数组」用 `T[]?`。元素是否可空由 `T?[]` 与 `T[]` 区分。空集合与 null 若业务不同，用类型或 API 二选一明确，避免混用。
- **与 general-coding 一致**：禁止仅因「担心」而加的冗余 null/长度检查；若担心成立，应改类型或修正赋值路径。
