# 无参计算成员优先用只读属性

- **何时用**：对外暴露「由当前状态推导出的值」、无参数、无副作用（或不把可观察副作用当作 API 契约）时，用 `public int Foo => ...` 或带 `get` 体的只读属性，而不是 `public int GetFoo()`。
- **常见形式**：`public int TeamSizeLimit => 5 + items.Sum(...);`
- **何时仍用方法**：需要异步（`Task`）、泛型实参依赖调用方、或团队约定「重计算 / 有副作用必须用方法」以区别于廉价属性时。
- **目的**：与 BCL / 惯用 C# 一致（如 `Count`、`Length`），调用点更短，也便于与 `object` 初始化器、部分绑定 API 配合。
