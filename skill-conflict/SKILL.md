# 技能冲突应对

应用或遵循多个技能前，先检查指令是否矛盾。

## 用户指示 vs 技能

如果用户的指示与任何技能说明的内容有冲突，也要立即停下并与用户说明。

说明时至少包含：用户相关指示要点、冲突的技能 `name`（及路径）、技能相关要点、为何不能在未确认前继续执行。不得擅自以技能覆盖用户指示，也不得擅自以用户指示覆盖技能而跳过说明。

## 优先级

当 project 技能与 global 技能冲突，以 project 技能为准。

| 类型 | 典型路径 |
|------|----------|
| project（项目） | 仓库内 `.cursor/skills/` |
| global（全局） | `~/.cursor/skills/`、`~/.agents/skills/` 等用户级技能目录 |

## 其他冲突

其他任何冲突，你必须直接指出两个技能说的有冲突。

向用户说明时至少包含：

1. **冲突双方**：技能 `name`（及路径若有助于定位）
2. **矛盾点**：各技能相关原文或要点（简短引用即可）
3. **当前采取**：若已按上节规则选定一方，写明依据；若无法自动裁决，暂停执行并请用户裁定

## 执行顺序

1. 核对用户指示是否与任一技能矛盾；若有，立即停下并说明
2. 列出本次任务将遵循的技能
3. 比对是否存在 project vs global 或其他两两矛盾
4. 按优先级执行，或先向用户报告冲突再等待指示
