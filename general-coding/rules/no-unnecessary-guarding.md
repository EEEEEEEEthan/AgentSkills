# 约定：杜绝非必要防御代码

## 规则

- **契约靠类型**：若成员在有效使用场景下不应为 null，应声明为不可空引用类型 `T`（或不可空值类型），而不是在每一处调用方写 `if (x != null)`。若确实可能缺省，声明为 `T?`，并在**真正需要分支**的边界（入口、序列化边界、用户输入）集中处理。
- **不写重复护栏**：构造函数、`Initialize`、反序列化等已保证的前置条件，业务逻辑里不再为同一条件叠 `!= null`、`Length > 0` 等「双保险」，除非规范或外部 API 明确要求。
- **例外**：与外部系统交互（网络、文件、第三方返回）、或编译器/分析器无法从类型表达的约束，保留最短必要校验。

## 不符合

```csharp
// `maps` 已声明为 `GameMap[]`（非 `GameMap[]?`），且开局流程保证赋值
if (game.maps != null && game.maps.Length > 0 && layer < game.maps.Length - 1) { ... }
```

## 符合

```csharp
// 信任 `GameMap[]` 契约；单层数边界用算术表达即可
if (layer < game.maps.Length - 1) { ... }
```

若某路径上 `maps` 可能尚未赋值，应在**该路径**修正生命周期或改为 `GameMap[]?` 并在唯一关口处理，而不是在下游到处判空。
