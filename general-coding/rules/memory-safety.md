# 约定：注意内存安全

## 规则

- **订阅要成对**：对长生命周期对象注册回调、观察器或句柄时，在订阅方或目标结束生命周期时**显式取消注册**（或交给有明确终点的容器托管），避免已析构/已释放对象仍被调用，导致未定义行为或“幽灵”逻辑。
- **引用生命周期**：在异步或回调中不要默认“对方一定仍有效”。若任一侧可能先结束，用弱引用、可空句柄、或把关系收束在明确所有者上，在单点做解绑与释放顺序约定。
- **循环引用**：引用计数、共享指针或互相缓存时，注意无意成环导致无法回收；需要回指时用弱引用、按需一方拥有另一方不拥有，或提供显式 `cleanup`/`teardown` 断环。
- **延迟销毁**：若存在“本周期末才真正释放”的语义，避免在已申报销毁的对象上再登记新异步工作或长寿命回调；解绑与判空策略应在项目内统一。

## 判断标准

新增长期回调、闭包捕获外部资源、或跨层缓存共享对象时，默认要问一句：**谁负责在何时解除引用 / 取消订阅？** 没有明确答案就不合并。
