# Skill Optimize

## 目标

在一次任务结束后，把执行中暴露的问题沉淀回技能，提升下次执行质量与一致性。

## 触发条件

当用户明确表达以下意图时启用：
- 优化技能
- 打磨技能
- 根据本次工作经验优化技能

## 执行步骤

1. 锁定输入信息：
   - 本次任务目标与结果
   - 用户明确指出的问题与不满点
   - 本次实际使用到的技能列表

2. 逐个复盘“本次使用过的技能”：
   - 哪条指令有效，保留
   - 哪条指令含糊、冲突或冗余，改写
   - 是否缺少触发词、边界条件、失败回退策略，补充

3. 只改“相关技能”：
   - 优先修改本次确实使用过的技能
   - 未使用技能不做泛化重写，避免噪音改动

4. 改写原则：
   - 保持简短、可执行、可检查
   - 优先给默认做法，减少分叉选项
   - 术语统一，避免同义词混用
   - 增加最小可用示例（输入 -> 输出）

5. 自检：
   - 是否明确 WHAT + WHEN（做什么 + 何时用）
   - 是否能直接指导下一次同类任务
   - 是否避免时效性表述与环境绑定硬编码

## 输出格式

输出时使用以下结构：

```markdown
本次使用技能：
- skill-a
- skill-b

发现的问题：
- 问题1
- 问题2

已做优化：
- skill-a：优化点...
- skill-b：优化点...

后续观察点：
- 观察项1
```

## 约束

- 必须基于“本次工作证据”优化，禁止空泛重构
- 必须覆盖“用户已指出的问题”，不能跳过
- 若信息不足，先提问补齐再改技能
